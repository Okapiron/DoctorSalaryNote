# 医師給与ノート roadmap

## 目的

このファイルは、「医師給与ノート」の開発フェーズと優先順位を管理する。

実装順序の判断材料として使い、各フェーズの完了条件を明確にする。

## Phase 0: 仕様整理

### Status

In progress

### Scope

- プロダクト文脈の整理
- MVP範囲の定義
- データモデルの概念整理
- 開発対象外の明確化
- 意思決定ログの作成

### Done

- docs/product_context.md がある
- docs/spec.md がある
- docs/data_model.md がある
- docs/roadmap.md がある
- docs/decision_log.md がある
- AGENTS.md がある

## Phase 1: MVP土台

### Scope

- SwiftUIアプリの土台
- SwiftDataによるローカル保存
- 下部タブ構成
- 勤務先管理
- 給与明細の登録、一覧、編集、削除

### Out of Scope

- ホーム集計
- 書類管理
- 分析画面
- Face ID / Touch IDロック
- CSV出力
- OCR
- クラウド同期

### Completion Criteria

- 勤務先を登録、編集、一覧表示できる
- 給与明細を登録、編集、一覧表示できる
- 給与明細が勤務先に紐づく
- アプリを再起動しても入力データが残る
- 入力必須項目のバリデーションがある

## Phase 2: ホーム集計

### Scope

- 年別の額面合計
- 年別の手取り合計
- 年別の控除合計
- 年度別表示
- 手取り率
- 直近の給与明細概要

### Completion Criteria

- 年別と年度別を切り替えて主要金額を確認できる
- 額面、手取り、控除、手取り率が表示される
- 支給年月に基づいて集計される

## Phase 3: 書類管理

### Scope

- 源泉徴収票
- 支払調書
- 給与明細ファイル
- 雇用契約書
- その他書類
- 年別の書類チェック

### Completion Criteria

- 書類を年別に登録できる
- 勤務先別に書類状況を確認できる
- 不足書類がわかる
- 添付ファイルを端末内に保存できる

## Phase 4: 分析

### Scope

- 年別収入推移
- 年度別収入推移
- 勤務先別収入
- 収入区分別収入
- 手取り率推移

### Completion Criteria

- 勤務先別、収入区分別に収入を比較できる
- 年別と年度別の推移を確認できる
- ユーザーが外勤収入や当直収入の変化を把握できる

## Phase 5: セキュリティと出力

### Scope

- Face ID / Touch IDロック
- CSV出力
- バックアップ
- プライバシーポリシー
- 免責事項

### Completion Criteria

- 生体認証ロックを有効化できる
- 年別CSVを出力できる
- 給与情報を外部送信しない方針が明記される
- 税務助言ではないことが明記される

## Phase 6: 入力補助

### Scope

- OCRによる入力補助
- 過去入力からの候補表示
- 勤務先別のよく使う収入区分
- 添付書類からの半自動登録

### Completion Criteria

- OCRは完全自動入力ではなく、入力補助として提供される
- ユーザーが確認、修正して保存する
- 誤認識時の修正がしやすい

## Long-term Ideas

- iCloud同期
- 複数端末同期
- 税理士共有用エクスポート
- 医師向け収入レポート
- 確定申告前チェックリスト
- ウィジェット
- Apple Watch対応

## Current Priority

現在の優先順位は以下。

1. 仕様管理ドキュメントを整える
2. MVPの入力体験を固める
3. 勤務先と給与明細のデータ構造を安定させる
4. 集計、書類、分析を段階的に追加する
