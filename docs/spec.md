# 医師給与ノート spec

## 目的

このファイルは、医師向け給与・収入管理アプリ「医師給与ノート」の機能仕様を管理するためのドキュメントである。

実装方針の詳細やコード構造ではなく、ユーザーに提供する体験、画面、操作、集計、制約を定義する。

## MVPの範囲

初回リリース相当のMVPでは、以下を対象とする。

- iPhone向けアプリとして利用できる
- データは端末内に保存する
- 下部タブで主要機能へ移動できる
- 勤務先を登録、一覧、編集、削除できる
- 給与明細を登録、一覧、編集、削除できる
- 給与明細に勤務先、支給年月、収入区分、額面、手取り、控除を記録できる
- 給与タブで、勤務先別に給与明細を絞り込みながら年月、勤務先、収入区分、金額を確認できる

MVPでは、集計や分析は最小限に留め、まずは継続的に入力できる収入台帳として成立させる。

## 主要タブ

MVP以降の完成形では、以下のタブ構成を想定する。

- ホーム
- 給与
- 勤務先
- 書類
- 分析
- 設定

MVPで実装するタブは、開発負荷に応じて絞り込んでよい。初回実装では「給与明細」「勤務先」を優先する。

## 勤務先管理

### 目的

給与明細や書類を勤務先単位で整理できるようにする。

### 登録項目

- 勤務先名
- 勤務先区分
- メモ
- 表示順
- 無効化状態

勤務先区分は以下を想定する。

- 常勤
- 外勤
- 当直
- スポット
- その他

### 基本操作

- 勤務先を新規登録できる
- 勤務先一覧を表示できる
- 勤務先情報を編集できる
- 勤務先を削除または無効化できる
- 給与明細に紐づく勤務先を選択できる

### 削除方針

給与明細に紐づいている勤務先は、物理削除ではなく無効化を優先する。

無効化された勤務先は、過去の給与明細では表示されるが、新規入力時の候補からは除外できるようにする。

## 給与明細管理

### 目的

勤務先ごとの給与・収入を月単位で記録し、後続フェーズの集計・分析・書類管理の基礎データにする。

### 登録項目

- 勤務先
- 支給年月
- 収入区分
- 額面金額
- 手取り金額
- 控除額
- 支払日
- メモ
- 添付ファイル

収入区分は以下を想定する。

- 常勤給与
- 賞与
- 外勤
- スポット
- その他

過去データや旧バージョンで登録された当直、日当直は、表示・集計上は外勤として扱う。

### 入力ルール

- 勤務先は必須
- 支給年月は必須
- 収入区分は必須
- 額面金額は必須
- 手取り金額と控除額は任意
- 金額は0以上の整数を基本とする
- 額面、手取り、控除の通貨単位は円とする

### 金額の扱い

額面金額は、支給総額または税引前の収入として扱う。

手取り金額は、実際に振り込まれた金額として扱う。

控除額は、給与明細上の控除合計として扱う。手取り金額と控除額のどちらか片方しか入力されないケースも許容する。

### 一覧と詳細

給与タブでは、勤務先別の給与状況を表示し、勤務先を選択するとその勤務先の給与明細だけに絞り込める。

給与明細一覧では、勤務先、支給年月、総支給額、添付書類の有無を簡潔に表示する。給与明細行の補助表示は、支給年月が切れずに読めることを優先し、「給与明細」などの固定語は繰り返し表示しない。

勤務先別の給与状況では、勤務先名、登録件数、合計総支給額を中心に表示し、直近支給年月や収入区分は表示しない。並び順は合計総支給額が多い勤務先を優先する。

給与明細をタップした場合は、まず読み取り専用の詳細画面を表示する。編集は詳細画面の編集ボタンから行う。

詳細画面では、総支給額、手取り、控除合計など、金額の意味が分かるラベルを明確に表示する。

編集画面でも、金額入力欄は項目名と入力金額を横並びにし、どの金額を入力しているか分かるようにする。

## ホーム

ホームは後続フェーズで実装する。

想定する表示内容は以下。

- 選択年または選択年度の額面合計
- 手取り合計
- 控除合計
- 手取り率
- 今月または直近支給分の概要
- 勤務先別の収入上位
- 書類不足アラート

## 書類管理

書類管理は後続フェーズで実装する。

対象書類は以下を想定する。

- 給与明細
- 源泉徴収票
- 支払調書
- 雇用契約書
- その他

書類管理は年別を基本軸とする。年度別切替は設けない。

書類は添付ファイルとして端末内に保存し、外部サーバーへ送信しない。

勤務先別の書類状況は、選択年の支給総額が多い勤務先から表示する。

勤務先別の書類状況カードをタップすると、その勤務先の登録済み書類に絞り込む。

源泉徴収票や支払調書が未登録・なしの場合は、年末整理で確認しやすいように注意色で表示する。ただし、支払調書はMVPでは必須判定しない。

## 分析

分析は後続フェーズで実装する。

想定する表示内容は以下。

- 年別収入推移
- 年度別収入推移
- 勤務先別収入
- 収入区分別収入
- 額面、手取り、控除の比較
- 手取り率の推移

勤務先別・収入区分別の内訳では、選択年の額面総額を基準に横棒の長さを決める。ただし、割合の数値やスケール説明は表示しすぎず、項目名、金額、件数、手取り額を優先する。

推移グラフでは、総支給額を棒、手取りを折れ線で表示する。インフォグラフィック的な装飾よりも、月・年、金額目盛り、横グリッド、直線的な推移が正確に読めることを優先する。目盛りは小数点や単位表記を省き、グラフ本体の表示幅を確保する。

推移グラフの手取り線は、給与明細がある月または年だけを結ぶ。未入力期間を0円の点として表示しない。

月別推移の縦軸は、年を切り替えても目盛りが大きく変動しないよう、登録済みデータ全体の月別最大額を基準にする。年次推移も登録済みデータ全体の年別最大額を基準にする。

分析画面の推移グラフ上では、左右スワイプで選択年を前後に切り替えられるようにする。

## 設定

設定では以下を扱う想定とする。

- 表示年度の切替
- Face ID / Touch IDロック
- CSV出力
- バックアップ
- プライバシーポリシー
- 免責事項

MVPでは設定機能は最小限でよい。

## 集計仕様

### 年別

1月1日から12月31日までを1年として扱う。

税務、源泉徴収票、支払調書、CSV出力、書類管理の基本軸とする。

### 年度別

4月1日から翌年3月31日までを1年度として扱う。

医師の勤務年度感覚で収入推移を見るための補助軸とする。

### 合計項目

- 額面合計
- 手取り合計
- 控除合計
- 手取り率

手取り率は、手取り合計 ÷ 額面合計で算出する。

額面合計が0の場合、手取り率は未表示または0として扱う。

## 添付ファイル

添付ファイルは後続フェーズで実装する。

想定する仕様は以下。

- PDF、画像を保存できる
- 給与明細または書類レコードに紐づける
- ファイル実体は端末内に保存する
- ファイル名、種別、作成日時を管理する

## セキュリティとプライバシー

- ログイン不要
- 端末内保存
- 外部サーバー送信なし
- クラウド同期なし
- Face ID / Touch IDロックは後続フェーズ
- 税務助言を行うアプリではないことを明示する

## 免責

本アプリは収入記録と書類整理の補助ツールであり、税務申告、税額計算、税務助言を目的としない。

税務判断が必要な場合は、税理士または税務署などの専門窓口に確認する。
