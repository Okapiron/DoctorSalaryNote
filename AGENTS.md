# AGENTS

## Project

医師向け給与・収入管理アプリ「医師給与ノート」。

このプロジェクトは、日本の医師が常勤、外勤、当直、スポット勤務、賞与などの収入を端末内で管理するためのiPhoneアプリを目指す。

## Current Scope

現時点の優先作業は仕様整理とMVP設計である。

実装に入る前に、以下のMarkdownドキュメントを優先して確認する。

- docs/product_context.md
- docs/spec.md
- docs/data_model.md
- docs/roadmap.md
- docs/decision_log.md

## Product Principles

- 税務申告ソフトではなく、収入台帳・書類整理アプリとして扱う
- 給与情報はプライバシー性が高いため、ローカル保存を基本にする
- MVPではログイン、サーバー保存、クラウド同期を行わない
- 年別は税務・書類管理の基本軸とする
- 年度別は収入推移確認の補助軸とする
- 初回実装では、勤務先管理と給与明細管理を優先する

## Development Boundaries

明示的に依頼されていない限り、以下は行わない。

- SwiftUI画面の新規作成
- Swiftモデルの新規作成
- プロジェクトファイルの編集
- Xcode設定の変更
- 依存関係の追加
- ビルド設定の変更

仕様管理タスクでは、Markdownファイルのみを編集する。

## Documentation Rules

- 仕様変更を行う場合は、関連するdocsファイルを更新する
- 重要な方針変更はdocs/decision_log.mdに記録する
- 実装詳細ではなく、プロダクト仕様、データ概念、判断理由を明確に書く
- 医師向けアプリであることを前提に、用語と入力項目を整理する

## MVP Reminder

MVPの中心は以下。

- 端末内保存
- 勤務先管理
- 給与明細管理
- 支給年月
- 収入区分
- 額面
- 手取り
- 控除

ホーム集計、書類管理、分析、Face ID、CSV出力、OCRは後続フェーズとして扱う。
